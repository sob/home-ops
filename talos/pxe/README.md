# Talos PXE server (Raspberry Pi)

A flash-and-go Raspberry Pi OS image that network-boots Talos on the metal
boxes, so you don't have to drag the KVM around the garage. It answers PXE
**only for the MACs defined in `talos/nodes/*`** and boots them into Talos
**maintenance mode**; you then apply the node config the usual way.

Nothing from this repo lives on the Pi — the config is baked into the image at
build time.

## How it works

```
metal box (UEFI netboot)
        │  DHCP/PXE broadcast
        ▼
Raspberry Pi  ── dnsmasq (proxyDHCP + TFTP) ──►  answers ONLY allow-listed MACs
        │        serves ipxe.efi (https, boot script embedded)
        ▼
iPXE  ── chain ──►  https://pxe.factory.talos.dev/pxe/<schematic>/<version>/metal-amd64
        ▼
Talos v1.11.2 (your schematic) boots to RAM → maintenance mode
        ▼
you:  just talos apply-config  (or talosctl apply-config)  →  installs to disk
```

- **proxyDHCP**: the Pi never hands out IP addresses — it only answers PXE, so it
  coexists with UniFi and touches nothing else on the LAN.
- **Allow-list = your node configs**: `render.sh` reads every `talos/nodes/*.yaml.j2`
  that has a real `hardwareAddr` and builds the dnsmasq allow-list. Nodes still on
  the `REPLACE_WITH_MAC` placeholder are skipped. Unknown MACs get no boot file
  (they can't netboot) but are logged, so a new box reveals its MAC.
- **Version/schematic** come from `../../.talos.env`, so netboot always matches
  what your machine configs install.

## Build the image (on your Mac)

Everything is a `just talos` command — no standalone scripts. Prereqs: macOS,
Docker (Desktop or Colima — used only to cross-build iPXE), and your SSH key.

```sh
just talos pxe-image
# overrides:
#   PI_IP=10.1.100.99/24 PI_GATEWAY=10.1.100.1 \
#   SSH_PUBKEY=~/.ssh/id_ed25519.pub  just talos pxe-image
```

Other recipes: `just talos pxe-macs` (show the allow-list), `just talos pxe-render`
(regen the config), `just talos pxe-build-ipxe` (just the loader).

Defaults baked in: static **10.1.100.99/24**, gw **10.1.100.1**, user **sob** with
your `~/.ssh/id_ed25519.pub`. Output: `talos/pxe/dist/talos-pxe.img`.

Flash with Raspberry Pi Imager (**Use custom** → the `.img`, no extra
customization) or `dd`. First boot installs dnsmasq and reboots (~1–2 min, needs
LAN internet once); after that it serves PXE fully self-contained.

## Prep the metal boxes (one-time BIOS)

- **Enable UEFI network/PXE boot** (IPv4).
- **Disable Secure Boot** (the iPXE loader is unsigned).
- Put **disk ahead of network** in the boot order — netboot should be the
  fallback, so a healthy node keeps booting from its installed disk. (Even if one
  did netboot by accident, Talos lands in maintenance mode and never auto-wipes.)

## Provision a node

1. Power the box on → it netboots → Talos comes up in **maintenance mode** and
   grabs a temporary DHCP IP.
2. Apply its config with your normal flow (`just` renders `talos/nodes/<ip>.yaml.j2`
   → `talosctl apply-config`). That sets the static IP + install disk and installs.

### A brand-new box whose MAC you don't know yet

1. Power it on (with PXE enabled). It'll fail to boot (not yet allow-listed) —
   but the Pi logs its MAC:
   ```sh
   ssh sob@10.1.100.99 'journalctl -u dnsmasq -f'   # watch for the new MAC
   ```
2. Copy an existing node file, set `hardwareAddr` (and disk serial, hostname, IP),
   then push the updated allow-list to the Pi without rebuilding the image:
   ```sh
   just talos pxe-render
   scp talos/pxe/dnsmasq.conf sob@10.1.100.99:/tmp/talos.conf
   ssh sob@10.1.100.99 'sudo install -m644 /tmp/talos.conf /etc/dnsmasq.conf && sudo systemctl restart dnsmasq'
   ```
3. Power-cycle the box — it netboots now.

## Updating

- **New node MAC** → the `scp dnsmasq.conf` one-liner above (no reflash).
- **Talos version bump** (`.talos.env`) → rebuild + reflash, since the version is
  embedded in `ipxe.efi`. (Or `scp` a freshly built `ipxe.efi` to
  `/srv/tftp/ipxe.efi` if you'd rather not reflash.)

## Files

The commands live in `talos/mod.just` (`pxe-macs`, `pxe-render`, `pxe-build-ipxe`,
`pxe-image`). This directory only holds the assets they use:

| file | purpose |
|------|---------|
| `Dockerfile.ipxe` | cross-builds `ipxe.efi` (amd64 UEFI, https, boot script embedded) |
| `firstrun.sh` | baked into the image; one-shot Pi first-boot (user/SSH, static IP, install + wire dnsmasq) |
| `README.md` | this file |
| `dnsmasq.conf`, `boot.ipxe`, `ipxe.efi`, `dist/`, `out/` | generated by the recipes (git-ignored) |
